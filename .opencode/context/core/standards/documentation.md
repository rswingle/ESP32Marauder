# Documentation Standards

Purpose: Ensure project documentation is clear, discoverable, and useful to both humans and agents.

Key points:
- Keep docs concise and task-focused
- Use examples that can be copy-pasted and adapted
- Document configuration and platform-specific steps (build/flash)
- Keep README.md as the single-entry onboarding doc

Example guidance:
1. Every public module should have a short README or header comment describing intent.
2. Use code blocks for commands and minimal runnable examples.

📂 Codebase References: CLAUDE.md (project-specific build/flash guidance) and README.md at repo root if present.
